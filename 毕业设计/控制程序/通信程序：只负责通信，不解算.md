##### DS102的通信配置和DIP设置
RS232C标准串行通信协议通过带有螺丝的大口和电脑连接；如果通过USB连接，则需要安装驱动（USB全程通用串行总线，所以插入的COM设备在设备管理器显示为COM）
	|DIP设置或许在USB下无效，因为驱动软件带有调整？
	单个DS102A可连接到2个轴，在驱动不超过6轴时，通过连接线串联三个DS102A，只连接到电脑的一个USB口（我甚至认为普通micro-usb线也能用），在此状态下：USB ID被定为0~3（根据DIP开关决定）
		DIP开关分别是（162页）2位Baud|2位LinkNumber（并联了几个？=>是的）|USB id|是否回报错误。
	[[plus/Pasted image 20250315034506.png]]
	Delimiter: 最后一位data，确定是0x0D，在通信中必须要传递到DS102. ASCII下，0x0d即\r，0x0a即\n
		每一个命令最后包括"\r"
		128到137页是所有可用命令，再到159页是命令细则，第159页传递可能错误返回（error_handle需要这些命令）
		！所有通信使用文本
	COM口的设置，是通过windows设备管理器找到对应的COM口完成的。&USB驱动就是为了能识别。
并联3个DS102（67页）需要专用线材“DS100-LINK2-0.5或DS100-LINK3-0.5”
	[[plus/Pasted image 20250315040232.png]]
	3个机器的开关决定了link number应该不同（分别是0,1,2），并且用2,1,0顺序通电或者同时通电
&关于为什么同一个驱动程序要两次安装：[reddit](https://www.reddit.com/r/techsupport/comments/4wfp1a/why_do_some_usb_devices_need_to_install_multiple/)
```
Windows refers to all driver instantiation as 'installation' to keep things simple for users.

A common example is having to 'install' multiple times for USB drives and devices that add (non-USB) ports to your computer. In these cases, Windows is simply creating a second instance of the driver that binds that port or volume to the new USB address of the device. Depending on the complexity of the driver, and whether there's already a cached instance or not, this might take some time.
```
